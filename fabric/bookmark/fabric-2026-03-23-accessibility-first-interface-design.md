# Accessibility-First Interface Design

[Original link](https://fabric.so/home?expandedFdocId=1ccc5580-5882-4f7a-b51e-029604c22dbf)

Accessibility-First Interface Design: a notepad entry about building projects with Google's NAI framework to adapt UI for accessibility needs from day one, including ADHD-friendly layouts, screen reader support, voice input, and progressive disclosure.

## Preview

Core Design Principle: Accessibility-First Principle All projects must be built accessibility-first using Google's Natively Adaptive Interfaces (NAI) framework. UI should adapt based on each user's accessibility needs — not bolted on after. What This Means ADHD simplification (reduced visual clutter, simplified layouts) Screen reader support baked in from day one Voice input as a first-class interaction mode Cognitive load management (progressive disclosure, chunked info) Sensory considerations (colour contrast, motion sensitivity) The interface itself reconfigures based on user needs Why Designing for edge users improves experience for everyone (curb-cut effect).
